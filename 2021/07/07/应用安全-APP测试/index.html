    1.测试四大组件

    模拟器上打开drozer

    物理机使用：adb connect 127.0.0.1:62001 去连接夜神模拟器，夜神模拟器默认端口是62001

    本地电脑上调用adb执行命令：adb forward tcp:31415 tcp:31415 进行端口转发

    正常情况下到drozer安装路径下运行 ：drozer console connect 会显示如下界面，之后即可正常运用drozer进行调试Android应用。

    1.1 获取包名

    drozer ：run app.package.list

    若是drozer 查看不到包名，可以使用adb进行查看：

    adb shell pm list packages

    1.2组件一Activity配置错误

    run app.activity.info –a 包名

    run app.activity.start –component 包名 路径组件名 启动它,若是绕过登陆看到app内部界面，则漏洞存在

    [示范]

    dz> run app.activity.info -a com.mwr.example.sieve

    Package: com.mwr.example.sieve

    com.mwr.example.sieve.FileSelectActivity

    Permission: null : null

    com.mwr.example.sieve.MainLoginActivity

    Permission: null

    com.mwr.example.sieve.PWList

    Permission: null

    这时我们可以通过run app.activity.start –component 路径包名 路径组件名 启动它

    [示范]

    dz> run app.activity.start –component com.mwr.example.sieve com.mwr.example.sieve.PWList

    1.3 组件二Content Provider配置错误，导致数据泄漏

    run scanner.provider.finduris -a 包名(返回数据显示Unable，则不可以访问)

    [示范]

    dz> run scanner.provider.finduris -a com.mwr.example.sieve

    Scanning com.mwr.example.sieve…

    Unable to Query content://com.mwr.example.sieve.DBContentProvider/

    Unable to Query content://com.mwr.example.sieve.DBContentProvider/Keys

    Accessible content URIs:

    content://com.mwr.example.sieve.DBContentProvider/Keys/

    1.4 组件三组件Service配置错误，导致非法权限提升

    run app.service.info -a 包名

    1.5 组件四Broadcast Receiver配置错误，导致拒绝服务、非法越权

    broadcast组件介绍

    在Android中，Broadcast是一种广泛运用的在应用程序之间传输信息的机制。而BroadcastReceiver是对发送出来的Broadcast进行过滤接受并响应的一类组件。广播接收者（BroadcastReceiver）用于接收广播Intent的, 广播Intent的发送是通过调用sendBroadcast/sendOrderedBroadcast来实现的。通常一个广播Intent可以被订阅了此Intent的多个广播接收者所接收。

    常见的静态广播定义示例；

    [示范]

    <intent-filter android:priority="1000">  

    <action android:name="android.provider.Telephony.SMS_RECEIVED"/>  

</intent-filter>

    利用drozer获取广播和进行拒绝服务攻击示例

    查看暴露的广播组件信息

    [示范]

    dz> run app.broadcast.info -a com.xxx.activity

    Package: com.xxx.activity

    com.xxx.lock.MyExitReceiver

    Permission: null

    com.xxx.lock.LockScreenReceiver

    Permission: null

    com.xxx.lock.LockScreenDeamonReceiver

    Permission: null

    查看暴露的广播组件详细信息

    [plain] view plain copy

    dz> run app.broadcast.info -a com.package.name -i

    dz> run app.broadcast.send –component 包名 –action android.intent.action.XXX

    利用空actoin和空extras拒绝服务

    2.代码安全

    2.1 反编译

    使用APKTOOL对APK包进行反编译，查看是否有混淆和加壳

    2.2 签名证书不规范

    keytool -printcent -file CERT.RSA

    2.3 Janus安卓签名漏洞

    java -jar GetApkInfo.jar APK包

    2.4 WebView漏洞

    源代码中搜索addJavascriptInterface，检查是否声明@JavascriptInterface来代替addjavascriptInterface。

    3.应用配置安全性

    3.1 开启allowbackup备份权限，存在备份数据泄露风险

    查看AndroidManifest.xml
    axmlprinter.py -i AndroidManifest.xml

    • 使用反编译软件JEB对apk进行反编译。

    • 在AndroidManifest.xml文件中查看allowBackup属性，检查是否未设置或者被设置为true。

    3.2 开启Debuggable属性，存在应用信息篡改泄露风险

    • 使用反编译软件JEB对apk进行反编译。

    • 在AndroidManifest.xml中搜索Debuggable属性，检查是否被设置为true。
